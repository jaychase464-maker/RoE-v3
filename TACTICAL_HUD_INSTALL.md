## Installation

1. Back up or commit the current project state.
2. Copy the package into the project root and allow Unity to import/compile. This hotfix replaces the earlier Tactical HUD scripts.
3. Exit Play Mode.
4. Run `Rules of Entry > Milestone 6B > Build Tactical HUD`.
5. Run `Tools > Rules of Entry > Milestone 6B > Validate Tactical HUD`.
6. Open `Assets/_Project/RulesOfEntry/Scenes/Prototype/ROE_Prototype.unity`.
7. Enter Play Mode.

The setup tool performs every prefab, scene, input, and component assignment. No Inspector assignment is expected.

## Controls

- MMB held: show command interface.
- MMB + `1`: Move.
- MMB + `2`: Hold.
- MMB + `3`: Stack.
- MMB + `4`: Open/Clear.
- MMB + `5`: Follow.
- MMB + `6`: Restrain.
- Release MMB without a number: close the interface without issuing an order.
- With MMB released, existing `1`, `2`, and `3` officer/team selection remains active for the current prototype.
- Existing cancel-order binding remains `Z`.

## Expected behavior

- The old mission/officer debug panels and all floating prototype status labels are absent from normal play.
- The idle weapon box is absent; temporary magazine-check/reload feedback still appears when relevant.
- Upper-left roster is a narrow, low-profile element list rather than a large dashboard panel, and displays every valid officer in the squad array rather than assuming exactly two.
- Each compact entry uses two lines: officer identity and qualitative condition above the current order and qualitative ammunition state.
- A thin blue edge/name marks the selected officer; the small condition pip uses green for normal, amber for attention, and red for critical.
- Health is qualitative: `FIT`, `WOUNDED`, `DOWN`, or `DECEASED`.
- Ammunition is qualitative: `GOOD`, `LOW`, or `CRITICAL`.
- The player still receives no exact ammo counter and must inspect their own magazine.
- Body-camera officer name, badge, department, battery, recording state, date, and time come from runtime data components.
- Body-camera chrome uses the approved rounded translucent shell, shield-shaped RoE mark, divider, recording dot, battery, and camera icons; it is not a generic debug rectangle.
- The command panel remains hidden until MMB is held.
- Door focus suggests Stack, subject focus suggests Restrain, and world-position focus suggests Move.
- Issued orders continue through the existing officer order state machines and ledgers.

## Common problems

- `OfficerCommandMenu` missing: rerun the Milestone 6B setup outside Play Mode and reimport `ROE_InputActions.inputactions`.
- Number key changes selection while MMB is held: confirm the player uses the updated `TacticalPlayerInput.cs` and input asset.
- HUD says references are incomplete: rerun setup; do not manually create an unconfigured HUD Canvas.
- Officer ammunition shows `UNKNOWN`: add/configure `OfficerAmmunitionStatus` through setup or the future NPC weapon system.
- Old command text overlaps the HUD: confirm `ROE_OfficerCommandDebugUI` is inactive in the operation scene.
- Body-camera name is the prototype placeholder: expected until campaign creation calls `BodyCameraIdentity.Configure`.
- Setup cannot find the player or squad: restore the completed Milestone 4 prototype before running Milestone 6B setup.

## Known boundaries

- The current operation has two officers, but the roster and team-order dispatch accept larger arrays.
- Individual direct-selection input beyond the first two officers will be finalized with the officer-management/element-selection system.
- NPC ammunition is a qualitative data feed until authoritative officer weapon simulation is implemented.
- The RoE body-camera shield is generated with uGUI text/line treatment; final brand art can replace it later without changing runtime data.
