# Temporary Suspect Character — Prototype Installation

## Goal

Use the supplied `SKM_Character.fbx` as the visible prototype suspect without replacing or weakening the working suspect AI, navigation, surrender, custody, injury, evidence, or hit-detection systems.

This is temporary presentation art. It is not the final Rules of Entry character pipeline.

## Install and apply

1. Complete `UI_PRESENTATION_INSTALL.md` and confirm Unity has no compiler errors.
2. Allow Unity to finish importing `Assets/_Project/RulesOfEntry/Art/Characters/Temporary/SKM_Character.fbx`.
3. Exit Play Mode.
4. Run `Tools > Rules of Entry > Temporary Characters > Apply Sample to Suspect`.
5. The tool configures the model as Humanoid, disables animation and material import, generates neutral HDRP/Lit materials, fits the character to 1.78 metres, and saves it inside the existing prototype suspect prefab.
6. The tool automatically runs `Validate Sample Suspect`. One performance warning is expected because the supplied mesh is high-density and has no LOD group. Errors are not expected.
7. Run all EditMode and PlayMode tests.

No package, project setting, Input Action, layer, tag, scene reference, or Inspector assignment is required.

## Play test

1. Enter the existing prototype operation.
2. Confirm the suspect appears as the supplied humanoid with neutral temporary materials instead of the colored primitive body.
3. Confirm the suspect still navigates and makes decisions normally.
4. Observe idle or patrol state: arms should rest near the body rather than remaining in a T-pose.
5. Let officers challenge the suspect: threatening/alert state should look guarded; surrendering/compliance should raise both hands.
6. Allow the normal initiative sequence to continue: kneeling, restrained, searched, and custody states should use the kneeling presentation while the existing custody timings and evidence remain unchanged.
7. Confirm player interaction, gunfire hit regions, officer challenges, automatic custody initiative, mission objectives, ROE review, and after-action scoring still work.
8. Confirm the Console remains clean.

The procedural poses are a bridge, not final animation. Sliding feet and imperfect hand placement are accepted temporary limitations; gameplay state transitions are not.

## Restore the original prototype visual

Exit Play Mode and run:

`Tools > Rules of Entry > Temporary Characters > Restore Prototype Suspect`

This removes the sample visual and pose driver, then re-enables the original primitive renderers. It does not delete the supplied FBX or generated materials.

## Common problems

### Unity cannot create a valid Humanoid avatar

Select `SKM_Character.fbx`, open the Rig tab, and inspect Configure Avatar for the first missing required bone. Do not force a Generic rig; the temporary pose bridge requires a valid Humanoid avatar. Report the first Console error and the unmapped bone name.

### The suspect remains a T-pose

Exit Play Mode, rerun Apply Sample to Suspect, then run Validate Sample Suspect. Confirm the saved suspect prefab contains a Humanoid Animator under `TemporarySampleCharacter` and one pose driver on the actor root.

### The suspect is invisible

Install the latest temporary-character hotfix and rerun Apply Sample to Suspect. The tool now enables the complete renderer hierarchy, expands skinned bounds, uses `AlwaysAnimate`, validates human-scale render bounds, and only hides the primitive fallback after the imported presentation passes. Any remaining failure should leave or restore a visible fallback and report the exact failed visibility condition in the Console.

### The model floats or drifts away from the suspect

Install Temporary Character Hotfix 2 and rerun Apply Sample to Suspect. The pose bridge now locks the fitted model-local transform and Humanoid body pose after each procedural muscle update. The actor root and NavMeshAgent may still move normally; only unintended motion inside `TemporarySampleCharacter` is blocked.

### The model is magenta

Confirm HDRP is active and the generated materials use `HDRP/Lit`. Rerun Apply Sample to Suspect after package import and shader compilation finish.

### Performance is poor

Use only one sample character. The source is approximately 82,000 vertices and has no LODs. Do not copy it to civilians or the officer squad. A production modular set needs optimized meshes, LODs, texture atlases or controlled material slots, and measured crowd budgets.

### AI or custody behavior changed

Run the Milestone 3, 4, and 5 validators immediately. The sample is presentation-only. If any protected actor component is missing, restore the prototype visual and report the first error before continuing.

## Stability gate

Do not accept the temporary-character integration until Unity compilation, its validator, all automated tests, the complete officer challenge-to-custody sequence, firearm hit response, mission completion, after-action review, and a clean Console all pass.
