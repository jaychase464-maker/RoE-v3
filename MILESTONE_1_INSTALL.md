# Milestone 1 Installation and Validation

This package installs the **Milestone 1 — First-Person and Tactical Interaction** integration candidate for Rules of Entry. It requires the completed Milestone 0 project and Unity `6000.5.2f1`.

## Scope

Milestone 1 adds:

- CharacterController first-person movement;
- mouse and gamepad look;
- walk, sprint, crouch, and obstruction-safe standing;
- cursor lock and gameplay-input switching;
- project-owned Input System action maps;
- reusable raycast interaction and hold interaction;
- interaction prompt UI;
- an animated prototype door and hold-to-use control panel;
- a repeatable graybox prototype builder;
- Milestone 1 validation and automated tests.

It does not add weapons, suspects, civilians, arrests, officer commands, objectives, scoring, or production art.

## Before importing

1. Confirm the repository is `C:\Users\Shadow\Videos\RoE v3` and `git status` is clean.
2. Close Unity.
3. Make a normal Git commit or branch if you have any uncommitted work.

## Import

Extract the contents of `RulesOfEntry-Milestone-1.zip` directly into the repository root. The extracted root must contain `Assets`, `README.md`, and this file—not another nested package folder.

The two existing assembly-definition files are intentional complete replacements:

- `Assets/_Project/RulesOfEntry/Runtime/RulesOfEntry.Runtime.asmdef`
- `Assets/_Project/RulesOfEntry/Editor/RulesOfEntry.Editor.asmdef`

They add references to the already-installed Input System and uGUI assemblies. No `Packages/manifest.json` or Project Settings file is included in the archive.

## Unity setup

1. Open the project with Unity `6000.5.2f1`.
2. Wait for script compilation and asset import to finish.
3. Confirm the Console has no red compiler errors.
4. Run **Tools > Rules of Entry > Milestone 1 > Build Gameplay Prototype**.
5. Allow the tool to open and save `ROE_Prototype.unity`.
6. Read the validation dialog and Console results.

The tool automatically:

- creates the `Player` and `Interactable` layers in free user-layer slots;
- creates four HDRP graybox materials;
- creates the player, door, control-panel, and interaction-UI prefabs;
- replaces only content previously generated by this Milestone 1 tool;
- disables the inactive template `Main Camera` root;
- builds the graybox room and places the player and interactables;
- assigns all required serialized references;
- saves the prototype scene and runs validation.

No manual Inspector assignment is required. Re-running the command is supported and rebuilds the generated content.

## Controls

| Action | Keyboard and mouse | Gamepad |
|---|---|---|
| Move | `W A S D` | Left stick |
| Look | Mouse | Right stick |
| Sprint | Left Shift while moving forward | Left-stick press |
| Toggle crouch | `C` | Right-stick press |
| Interact | `E` | West/left face button |
| Release/recapture cursor | Escape | Start |

## Manual smoke test

1. Open `Assets/_Project/RulesOfEntry/Scenes/Prototype/ROE_Prototype.unity`.
2. Clear the Console, enter Play Mode, and click the Game view if needed.
3. Walk and look around; verify walls and props block the player.
4. Hold sprint while moving forward and verify the speed increase.
5. Press crouch, pass through the low blue tunnel, and verify standing is blocked while beneath it.
6. Aim at the training door and press Interact; verify the door opens and its prompt changes to Close Door.
7. Aim at the wall control panel and hold Interact; verify the progress bar fills and the indicator changes state.
8. Press Escape; verify the cursor releases and player movement/look stop. Press Escape again to resume.
9. Exit Play Mode and confirm the Console contains no errors or exceptions.

## Automated tests

Open **Window > General > Test Runner**:

1. Run all Edit Mode tests.
2. Run all Play Mode tests.
3. Confirm both existing Milestone 0 tests and the new Milestone 1 tests pass.

Also run **Tools > Rules of Entry > Milestone 1 > Validate Gameplay Prototype** after any scene or prefab change.

## Common problems

### Input System or UI namespace compiler error

Confirm the two package asmdef files replaced the existing versions and that `com.unity.inputsystem` `1.19.0` and `com.unity.ugui` `2.5.0` remain installed.

### Input asset or prefab missing

Confirm the archive was extracted into the repository root, wait for import to complete, then run the Milestone 1 setup command again.

### No free layer slot

The tool never overwrites a named layer. Free one user layer in **Edit > Project Settings > Tags and Layers**, then rerun setup.

### Pink graybox materials

Confirm HDRP is active through `HDRP High Fidelity`, then rerun setup so the materials are rebuilt with `HDRP/Lit`.

### Two cameras or AudioListeners

Do not manually enable the original template `Main Camera`. Rerun setup, then run Milestone 1 validation.

### Mouse remains visible or look does not respond

Click the Game view and press Escape once. Confirm the Console is clean and the player prefab uses `ROE_InputActions.inputactions`.

## Stability gate

Do not mark Milestone 1 stable or push it as completed until all of these are confirmed:

- Unity compiles with zero errors;
- Milestone 1 validation reports zero errors;
- all Edit Mode and Play Mode tests pass;
- the manual smoke test passes;
- Play Mode produces no errors or exceptions.
