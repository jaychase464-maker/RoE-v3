# Milestone 6A — Headquarters and Rugged Operation Planning

## Goal

This milestone establishes the intended campaign deployment loop:

`Main menu → playable PD headquarters → physical mission selection → rugged planning tablet → ready up → operation`

The PD is a gameplay-first greybox. It contains labeled future areas for the loadout cage, officer management, shoot house, operations, and specialized-support staging without claiming those future systems are complete. The tablet presentation follows the approved concept with a broad rugged bezel, side function keys, seven top tabs, two information columns, briefing status, and a persistent Start Operation area.

## Install

1. Confirm the current repository is committed and pushed.
2. Exit Play Mode and close Unity.
3. Extract `RulesOfEntry-Milestone-6A-Headquarters-Tablet.zip` directly into `C:\Users\Shadow\Videos\RoE v3`.
4. Merge folders and replace files when Windows asks.
5. Reopen Unity `6000.5.2f1` and wait for compilation and asset import.
6. Confirm the Console has no compiler errors.
7. Run `Tools > Rules of Entry > Milestone 6A > Build Headquarters and Tablet Prototype`.
8. Let the tool create the planning asset and headquarters scene, rebuild the front end, configure Build Settings, and run validation.
9. Run Test Runner > EditMode > Run All.
10. Run Test Runner > PlayMode > Run All.

The setup is safe to rerun. No Inspector assignment, Input Action, tag, layer, package, or 3D model is required.

## Expected boundary

The selected entry ID is preserved, but the existing operation scene does not yet contain authored entry spawn anchors. Milestone 6B will connect each stable entry ID to validated scene anchors and team formations. This milestone deliberately does not teleport the player to guessed coordinates.

K9, drone, tactical medic, negotiator, officer careers, loadout editing, and the shoot house are represented only by stable planning or headquarters boundaries. They are not claimed as implemented gameplay.

## 3D models

No model is required to install the hardware-only tablet correction. It matches the supplied rugged-device direction without including hands or a baked environment, but it remains a flat transparent overlay. Replacing it with a viewpoint-correct physical object will require a rugged tablet/MDT model with a separate emissive screen mesh, shell and rubber guards, hardware buttons, correct pivots, and PBR textures. First-person hands are not part of the current desired presentation and will not be required unless that direction changes. Before final-art replacement, the likely environment categories are:

- modular police-station interior shell, corridors, doors, stairs, and trim;
- operations-room desks, displays, briefing boards, radios, and chairs;
- armory cage, weapon racks, lockers, benches, and equipment storage;
- office desks, filing/storage, evidence-room shelving, and station signage;
- modular shoot-house walls, catwalks, doors, observation glass, and range fixtures;
- rugged Toughbook/tablet with separate screen, keyboard/lid, hinge, and correct pivots.

Exact dimensions and quantities will be requested only after the greybox layout and interaction routes pass.

## Common problems

### Setup stops at Milestone 5

Run the Milestone 5 validator and fix its first error. Milestone 6A preserves the existing suspect, civilian, room, officer, ROE, and after-action targets.

### Operations says headquarters is unavailable

Exit Play Mode and rerun the Milestone 6A setup. Confirm Build Settings begins with Front End, Headquarters, then Prototype.

### Headquarters reports incomplete Milestone 4 squad references

Install Milestone 6A Hotfix 2, exit Play Mode, and rerun the Milestone 6A setup. The player prefab keeps its Milestone 4 command layer, but the generated headquarters scene must save that operation-only component disabled. Do not rerun Milestone 4 to fix a headquarters-only error.

### Mission terminal has no prompt

Confirm the player is within three meters and aiming at the terminal. Rerun setup if `M6A_MissionAssignmentTerminal` or the `Interactable` layer is missing.

### Tablet opens but buttons do not respond

Confirm the headquarters contains one EventSystem with `InputSystemUIInputModule` and no `StandaloneInputModule`. Rerun setup rather than manually adding another EventSystem.

### Ready Up is disabled

Assign at least one available officer and select a valid entry. Specialized support is not required in Milestone 6A.

## Stability gate

Do not close Milestone 6A until clean compilation, both validators, all tests, the complete headquarters/tablet/deployment smoke test, Milestones 1–5 regressions, and a clean Console pass.
