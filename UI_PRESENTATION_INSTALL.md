# Milestone 5.5 — Front-End and UI Presentation

## Goal

This delivery gives Rules of Entry a deliberate first impression and one coherent interface language without disturbing the gameplay systems that already pass.

The authored flow is:

`Trooper Studios splash → photosensitivity/legal warning → Rules of Entry title → main menu → loading → playable prototype`

## Install

1. Confirm the current repository is committed and pushed.
2. Exit Play Mode and close Unity.
3. Extract `RulesOfEntry-Frontend-UI-6.zip` directly into `C:\Users\Shadow\Videos\RoE v3`.
4. Merge folders and replace files when Windows asks.
5. Reopen Unity `6000.5.2f1` and wait for compilation and asset import to finish.
6. Confirm the Console has no compiler errors.
7. Run `Tools > Rules of Entry > UI Presentation > Build Front End and Restyle HUD`.
8. Allow the tool to validate Milestone 5, generate and save the front-end scene, restyle the prototype HUD, configure Build Settings, and run UI validation.
9. Run all EditMode and PlayMode tests.

No package, Input Action, tag, layer, audio asset, or Inspector assignment is required for the UI. The ZIP includes all three UI textures and its licensed condensed typeface. Temporary-character setup is a separate, reversible step documented in `TEMPORARY_CHARACTER_INSTALL.md`.

The setup is safe to rerun. It regenerates the front-end scene and presentation-owned prototype objects, then reapplies presentation overrides to the existing functional HUD instances.

None. The setup tool creates and connects all serialized references. Do not manually add a second EventSystem or UI input module.

## Player Settings

The tool sets:

- Company Name: `Trooper Studios`
- Product Name: remains `RoE v3`
- Default resolution: `1920 × 1080`
- Fullscreen mode: borderless full-screen window

Unity's own legal engine splash may appear before the Trooper Studios scene depending on license and platform requirements. The Trooper Studios screen is the first authored game scene content.

## Front-end test

1. Open `Assets/_Project/RulesOfEntry/Scenes/FrontEnd/ROE_FrontEnd.unity` if setup did not leave it open.
2. Enter Play Mode.
3. Expected: the supplied Trooper Studios splash fades in, holds briefly, and fades out.
4. Expected: the photosensitivity/legal screen appears and remains indefinitely.
5. Confirm ordinary keys and mouse clicks do not dismiss the warning.
6. Press Enter, numpad Enter, or controller A/South and confirm the title appears.
7. Confirm the title accepts any keyboard key, left mouse click, controller Start, or controller South button.
8. Confirm the main menu uses flat text navigation with no boxed cards or decorative tactical grid.
9. Confirm Continue Campaign and New Campaign are dim and cannot be selected or activated.
10. Confirm mouse hover and keyboard/controller selection shifts the selected label slightly and reveals a narrow blue focus bar.
11. Open Settings, change volume, toggle fullscreen, cycle quality, and return with Back or Escape/controller East.
12. Open Credits and return.
13. Choose Operations and confirm the loading screen identifies `TRAINING OPERATION: CONTROLLED RESOLUTION`, reports the scene destination and live percentage, reaches Destination Ready, and loads `ROE_Prototype.unity`.
14. Repeat with Training; its context label changes to `TRAINING`, while the actual mission name remains sourced from the same Milestone 5 mission asset during Milestone 5.5.

## Persistence test

1. Change all three settings.
2. Exit Play Mode.
3. Enter Play Mode again and reach Settings.
4. Expected: volume, fullscreen, and quality match the saved values.

These prototype settings use `PlayerPrefs`; production work should later add explicit apply/cancel behavior, resolution/refresh-rate selection, and a versioned settings service.

## Gameplay HUD regression

After Begin Operation loads the prototype:

1. Confirm normal movement, look, interaction, weapon, mission, officer, AI, surrender, custody, room-clearance, and after-action behavior still work.
2. Confirm the tactical interaction prompt appears at the lower center when aiming at an interactable.
3. Confirm the manual weapon state remains at the lower right and does not expose an exact ammunition count.
4. Confirm mission status is upper left and officer status is upper right.
5. Press F10 and confirm AI diagnostics appear; press F10 again and confirm they hide.
6. Complete the peaceful mission and confirm the final after-action report still reaches the expected result.
7. Confirm the Console remains clean.

## Common problems

### Setup stops at the Milestone 5 baseline

Run the Milestone 5 setup and validator first. Fix the first reported error, then rerun UI Presentation setup.

### Buttons do not respond

Exit Play Mode and rerun UI Presentation setup. Confirm the front-end scene contains one `EventSystem` with `InputSystemUIInputModule` and no `StandaloneInputModule`.

### Begin Operation says the scene is unavailable

Rerun setup and confirm Build Settings starts with enabled `ROE_FrontEnd` and enabled `ROE_Prototype`.

### Rules/Entry or the loading destination text is missing

Exit Play Mode and rerun UI Presentation setup. Revision 4 rebuilds the title rectangles, enables vertical overflow for oversized labels, and reconnects the mission-definition destination reference. Do not type the mission name into the background image.

### The gameplay HUD disappeared

Press F10 only affects the AI diagnostic panel. If another panel is missing, first confirm its normal gameplay condition is active. Then exit Play Mode and rerun setup to restore saved presentation overrides.

### The Trooper Studios screen is not the very first visible logo

Unity may require its engine splash before scene content. The authored Trooper Studios screen begins when `ROE_FrontEnd` loads.

### UI is clipped at an unusual aspect ratio

The canvas scales from 1920×1080 and is designed to remain usable across common 16:9 and 16:10 displays. Ultrawide and 4:3 safe-area polish remains a live validation item; report the resolution and attach a screenshot if clipping occurs.

## Stability gate

Do not close Milestone 5.5 until compilation, setup, UI validator, all tests, front-end/controller/settings/load flow, Milestones 1–5 regressions, and a clean Console pass in Unity.
